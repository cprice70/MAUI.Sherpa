## 🏔️ v0.7.0 — "Calibrating the Instruments"

Up at altitude, conditions change fast. A climber who can't read the mountain — temperature shifts, barometric drops, how hard the body is working — doesn't last long above the clouds. This release straps a full instrument panel to your wrist: real-time app profiling with CPU tracing and memory analysis, a built-in trace viewer, and a new Xcode management dashboard. We've also packed a command-line radio so AI agents and shell scripts can talk to Sherpa directly. The gear is getting serious.

### 🆕 New Features

- **App Profiling — CPU Tracing & GC Dumps** — The marquee feature of this release. A full profiling pipeline for .NET MAUI apps: guided capture wizards for `dotnet-trace` and `dotnet-gcdump` across Android, iOS, Mac Catalyst, macOS, and Windows. Sherpa handles dsrouter setup, environment variables, port forwarding, and all the fiddly ceremony so you can focus on finding the hot paths. (#92, #105)
- **Speedscope Trace Viewer** — Captured `.nettrace` files are converted to Speedscope format and displayed in an embedded viewer — flame graphs, time-ordered views, and left-heavy aggregation without leaving the app. No more copying files to a web browser.
- **GC Dump Viewer** — A dedicated viewer for `.gcdump` files that parses the GC heap report and presents object type breakdowns, instance counts, and byte sizes in a sortable, searchable table.
- **Profiling Session Library** — All captured traces and dumps are organized into sessions with metadata, timestamps, and quick-access cards. Re-open any previous capture for analysis without hunting through directories.
- **Xcode Management** — Browse installed Xcode versions, switch the active `xcode-select` path, and download new versions from the Apple Developer portal — all from a single page. No more `sudo xcode-select -s` in the terminal. (#99)
- **iOS Simulator Permissions Inspector** — A new tab in the Apple Inspector that lists, grants, resets, and revokes privacy permissions (camera, location, contacts, photos, etc.) for any booted simulator. (#90)
- **maui-sherpa CLI** — A new command-line tool (`maui-sherpa`) for headless environments and AI agents. Covers Android SDK, emulators, keystores, Apple profiles, simulators, Xcode, Doctor, and workloads — all with structured JSON output and MCP-style tool discoverability. (#91)
- **MauiDevFlow Platform Tab** — A new inspector tab that surfaces platform-level data from the connected device: battery, connectivity, display, storage, sensors with live streaming, location, and app preferences.
- **Profiling Copilot Skill** — A bundled Copilot skill that gives the AI assistant context about .NET profiling tools, trace formats, and common performance analysis patterns.

### ✨ Improvements

- **DevFlow Profiling Tab** — The inspector gained a dedicated profiling tab for orchestrating captures against a connected agent, with platform-aware configuration and real-time progress.
- **Multi-Sensor Streaming** — The platform tab supports streaming multiple device sensors simultaneously (accelerometer, gyroscope, magnetometer, barometer, compass) with inline live values and a "Stop All" button.
- **Keystore Import Password Prompt** — Importing an Android keystore now prompts for the password immediately, rather than silently failing to read alias metadata. (#101)
- **Workload Fix Actions** — The Doctor page now offers one-click fix actions for missing or outdated .NET workloads, and the empty-state message no longer clips its container. (#97)
- **Quick Start Guide Link** — The DevFlow "Get Started" hint now actually opens the Quick Start guide in your browser instead of doing nothing. Inline code styling flows naturally with the surrounding text. (#112)

### 🐛 Bug Fixes

- **Publish Key Mappings Ignored** — Publishing secrets used the raw source key names instead of the mapped destination names configured in the profile. A mismatch between label-based and enum-based key prefix generation meant mappings were silently skipped. (#111)
- **Publish Repo ID Format** — Publishing to GitHub Actions failed because the numeric repository ID was stored instead of the `owner/repo` format that the API expects. (#111)
- **Publish NullReferenceException** — After a successful publish, the wizard would throw a NullReferenceException when trying to read an unset result. (#111)
- **Xcode Switch Badge** — Switching the active Xcode version didn't update the "Active" badge in the UI until a full page refresh. (#106)
- **Keystore Cloud Deletion** — Deleting a cloud-synced keystore could fail silently, and importing on macOS didn't prompt for the password. (#104)
- **Mac Catalyst Profile Devices** — The provisioning profile device list was showing iOS devices for Mac Catalyst profiles instead of filtering to Mac-only devices. (#103)
- **Bundle ID Capability Toggle** — Non-API-toggleable capabilities could be clicked in the UI, causing confusing error states. They're now properly disabled. (#48)
- **Windows Black Screen** — The app launched to a black screen when installed under `Program Files` on Windows due to a path resolution issue. (#93)
- **Mac Catalyst Network Entitlement** — Added the missing `NSLocalNetworkUsageDescription` to the Info.plist so the MauiDevFlow agent can bind its local HTTP listener. (#96)

### 🔧 Infrastructure

- **MauiDevFlow 0.23.0** — Bumped the minimum required agent version to 0.23.0, bringing platform info APIs, sensor streaming, preferences management, and on-device profiling support. (#109)
- **Profiling Sample App** — A new `MauiSherpa.ProfilingSample` project with intentional CPU, memory, and allocation scenarios for testing the profiling pipeline end-to-end.
- **Comprehensive Profiling Tests** — 100+ new unit tests covering the profiling catalog, prerequisites, capture orchestration, artifact analysis, artifact library, and context services.
- **Copilot Profiling Tools** — New Copilot tool registrations for profiling operations: checking prerequisites, browsing the capture catalog, planning captures, and analyzing artifacts.
- **Homebrew Install Fix** — Corrected the `maui-sherpa` command name in the Homebrew installation docs. (#94)
