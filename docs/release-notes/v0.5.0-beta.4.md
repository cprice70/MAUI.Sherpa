## 🏔️ v0.5.0-beta.4 — "Stocking Camp III"

Camp III is established, and the team is hauling up supplies. This release is all about the logistics of getting secrets where they need to go — a new Publish Profiles feature lets you define exactly which certificates, keystores, and secrets to publish, to which repositories, and review everything before committing to the push. Think of it as labeling every oxygen canister and clipping it to the right rope line.

### 🆕 New Features

- **Publish Profiles** — A brand-new page for creating reusable publish profiles that bundle Apple certificates, Android keystores, managed secrets, and key mappings into a single publishable unit. Your packing list for the summit push.
- **Apple Config Wizard** — A multi-step wizard inside the profile editor walks you through selecting an Apple identity, platform, distribution type, signing certificate, installer certificate, provisioning profile, and notarization credentials. Each step filtered to only show what's compatible.
- **Multi-Publisher Destinations** — Profiles now support multiple publish destinations (GitHub, GitLab, Azure DevOps, etc.) instead of a single repository. Add as many targets as you need from a slick provider → repository picker dialog.
- **Publish Review Dialog** — Clicking "Publish" opens a 3-step review wizard: select which secrets to include, confirm destinations (with inline add), then watch live per-destination publish status as operations complete.
- **Certificate Sync Status Badges** — The certificate picker now shows sync status on every cert: *Synced* (keychain + cloud), *Local* (keychain only), *Cloud* (cloud only, not selectable), or *No Key* (unavailable). Only certificates with a local private key can be selected.

### ✨ Improvements

- **Key Mappings Redesign** — Secret key mappings are now always visible in a clean row layout with inline editable destination fields, replacing the old collapsible `<details>` sections. Wider inputs, separator lines between rows, and a consistent add/remove button column.
- **Secret Picker Dialog** — The managed secrets picker is now a centered modal overlay with a scrollable body, replacing the inline card that pushed content around.
- **Config Card Styling** — Apple and Android config cards use transparent headers with bottom-border separators instead of tinted backgrounds. Badges moved to the right of titles. Content padding refined.
- **Profile Save Indicator** — Saving a profile now shows a spinner overlay inside the modal so you know the cloud sync is working, instead of closing the dialog and leaving you wondering.
- **Resolve Secrets Inside Dialog** — The publish review dialog resolves secrets internally with a progress spinner, rather than resolving before opening. Partial failures show a warning banner listing which items couldn't be fetched.

### 🐛 Bug Fixes

- **Apple Identity Not Set During Resolve** — `ResolveSecretsAsync` now loads the Apple identity stored in each config's `IdentityId` and sets it on the identity state service before making API calls. Previously it relied on whatever identity happened to be active in the app, which was often null.
- **Wizard Content Clipping** — Fixed wizard step content getting clipped at dialog corners.
- **Create Profile Card Visibility** — Fixed list cards blending into the dialog background in both light and dark themes.
- **Wizard Search Spacing** — Corrected dead CSS selectors that broke the gap between search inputs and result lists.

### 🔧 Infrastructure

- **CI Pipeline Cleanup** — Removed Mac Catalyst from the CI build pipeline, streamlining builds to the active platform targets.
